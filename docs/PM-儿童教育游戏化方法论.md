## 一、用一句话管住产品

> **每天 5 分钟，用一道母题类型打败今日 Boss，把战绩晒到班里。**

| 维度 | 定义 |
|------|------|
| 用户 | 小学 2～4 年级 |
| 英雄场景 | 每日 Boss 战（全产品只推一条主链路） |
| 学习单元 | **母题类型**（一类题），不是题海、不是单个算式 |
| 传播物 | 战绩海报（图 > 字） |
| 集体动机 | 班级 Boss 击败数 + 领地 |
| 不做 | 充值、战力掠夺、长剧情、无节制加页面 |

## 二、核心公式（可复用到任何儿童教育产品）

### 2.1 「酷」的公式

```
酷感 = 身份认同（我在打谁）× 即时反馈（数字/震动/动画）× 社交货币（可晒物）
```

**自检**：缺任一环，都会出现「功能很多却不酷」。

| 环 | 孩子心里的问句 | 产品要给的答案 |
|----|----------------|----------------|
| 身份认同 | 我在打谁？ | 今日外号 + 母题类型（不是「第 3 题」） |
| 即时反馈 | 我打中了吗？ | 30 秒内有伤害、连击、弱点、必杀 |
| 社交货币 | 我能显摆什么？ | 海报图 + 分享标题 + 同学可接力 |

### 2.2 「好记、愿聊、家长信」的公式

```
记忆 = 今日怪名（题型人设）+ 母题一句话（类型摘要）
信任 = 海报上保留母题类型一行（家长/老师可核对）
传播 = 同学聊怪名，家长看类型，不是聊分数
```

### 2.3 情绪曲线（5 分钟仪式）

```mermaid
flowchart LR
  open[打开] --> gate[通缉令预告]
  gate --> fight[3击Boss战]
  fight --> weak[错答点亮弱点]
  weak --> peak[母题必杀]
  peak --> remember[记住两个词]
  remember --> poster[战绩海报]
  poster --> classGroup[班级群接力]
```

每一环目标 **< 30 秒**；多一步规则说明即流失。

## 三、方法论：从「功能多」到「愿天天玩」

### 3.1 根因诊断（经验）

| 表象 | 常见误判 | 真实根因 |
|------|----------|----------|
| 像作业不像游戏 | 缺功能 | 缺**英雄场景密度**，不是缺模块 |
| 孩子不分享 | 缺分享按钮 | 缺**可晒物**（海报图） |
| 记不住学了啥 | 题太少 | 绑错对象（绑算式不绑**题型**） |
| 产品负责人乏力 | 方向错了 | 用**自检清单视角**打开，而非孩子视角 |

### 3.2 五维升级框架（决策时不偏科）

| 维度 | 问句 | 本项目抓手 |
|------|------|------------|
| 更酷 | 3 秒知道打谁、30 秒有爽感？ | 通缉令、连击、APNG、必杀峰值 |
| 更好玩 | 其它模块是否**服务 Boss**？ | 竞技场复仇、抽卡 tag、错题=弱点研究 |
| 更有效 | 记住的是**类型**还是**数字**？ | 三段式题组、教战宝、错题间隔重复 |
| 更特色 | 和作业 App、纯游戏差在哪？ | **母题日报** + 题型外号 + 零氪 |
| 更传播 | 海报能否变成同学接力？ | 外号标题、挑战深链、班级真实数据 |

**原则**：五维都指向同一条主链路，不各推一条线。

### 3.3 低年级认知约束（设计红线）

| 约束 | 做法 |
|------|------|
| 认字有限 | 按钮 < 6 字；说明 = 图标 + 1 句 |
| 耐心 < 30 秒/环 | 3 题强反馈 > 30 题平淡 |
| 错答心理 | 「Boss 弱点」替代红叉羞辱；可同题再打 |
| 峰值必须存在 | 第 3 题全屏「母题必杀」=「赢了一局」 |

### 3.4 学玩分离（家长信任）

| 机制 | 目的 |
|------|------|
| Gate 先 Boss 后进主世界 | 先学后玩 |
| 主世界玩法绑定今日母题 | 学完能复仇、能抽对口卡 |
| 海报保留母题类型 | 传播物 = 信任锚点 |
| 不做战力榜、邀请返现 | 班级荣誉 > 攀比 |

## 四、方法论：母题类型 Boss 命名（本次沟通核心纠正）

### 4.1 一句话原则

> **外号绑「一类题」，不绑「一道算式」。**

| ❌ 错误 | ✅ 正确 |
|--------|--------|
| `84÷7` →「八四分七怪」 | **切刀问题** →「**切猪肉怪**」 |
| 孩子记数字谜语 | 孩子记「切猪肉怪 + 切几段要几刀」 |
| 同学无话题 | 「今天我们班打切猪肉怪！」 |

**标准范例（沟通中确立）**：

- 类型：切香肠要几刀（段数 − 1）  
- 外号：切猪肉怪  
- 例题：切 8 块几刀、切 18 块几刀（数字在例题里，不在外号里）

### 4.2 双名体系

| 层 | 字段 | 作用 |
|----|------|------|
| 传播层 | `dailyNickname` | 聊天、分享、通缉令主标题 |
| IP 层 | `name`（系列 Boss） | 立绘、图鉴、视觉延续 |
| 教研层 | `topicTypeLabel` + `motherTopicSummary` | 家长/老师识别题型 |

详见 [PM-母题类型Boss命名.md](./PM-母题类型Boss命名.md)。

### 4.3 A1 + A3 协作（内容生产）

| 阶段 | 做法 | 产出 |
|------|------|------|
| A1 | 教研列母题类型清单 + 人工首版外号 | `bossNicknames.json` |
| A3 | 大模型按**类型**批量生成 → 人工抽检 | 同文件更新 |
| 上线 | `mergeBossNickname` 合并到每日 Boss | 全链路统一外号 |

**禁止**：在线每日自动生成外号（未抽检前不上线）。

## 五、方法论：需求选择与排期

### 5.1 功能清单 → 可选实现表

对大项拆编号（如 A1～A8、B1～B6），标注：

- 价值（对孩子/传播/教育）
- 工作量（小/中/大）
- 依赖（后端/教研/美术）

**让负责人「勾选」而非「全做」**，避免共识模糊。

### 5.2 最小闭环优先（本次选定）

在验证假设前，**只打包最小集合**：

| 编号 | 内容 | 验证假设 |
|------|------|----------|
| A1+A5+A6 | 词库 + 双名 + 全链路外号 | 好记 |
| B2 | Gate 通缉令 | 3 秒懂打谁 |
| B3 | 连击 UI | 30 秒有爽感 |
| E6 | 分享标题用外号 | 愿晒 |

**未选入最小闭环的**，等访谈通过再开（如教战宝、异步 PK、真实班级数据）。

### 5.3 验证先于加功能（第 0 步）

| 指标 | 目标 | 方法 |
|------|------|------|
| 外号+母题复述 | ≥ 4/5 | 通关后 3 问访谈 |
| 分享点击率 | ≥ 15% | 结算页埋点/观察 |
| 班级群话题 | 出现怪名讨论 | 运营观察 1 周 |

**不达标时**：先改 `bossNicknames.json` 或海报，**不**先加新系统。

### 5.4 三周节奏模板（可复用）

| 周 | 主题 | 典型交付 |
|----|------|----------|
| 第 1 周 | 爽感/仪式 | 通缉令、连击、APNG、弱点条 |
| 第 2 周 | 教育闭环 | 教战宝、错题→明日热身、图鉴 |
| 第 3 周 | 传播真数据 | 挑战深链、班级真实通关、小程序码 |

## 六、方法论：体验包 A / B / C（排期参考）

沟通中形成的**三选一扩展包**，避免并行摊大饼：

| 包 | 主题 | 典型项 | 何时优先 |
|----|------|--------|----------|
| **A 更酷** | 爽感密度 | Boss APNG、弱点条 | 访谈通过但觉得「不够酷」 |
| **B 更牢** | 教育闭环 | 教战宝、错题明日热身 | 复述率 OK 但理解不深 |
| **C 更传播** | 同学接力 | 挑战深链、异步 PK 文案 | 愿晒不足 |

**包 A 已落地（2026-05-28）**：

- `BossPortrait`：APNG 自动探测，无则 PNG + 呼吸动效  
- `WeaknessBar`：答错点亮，同题再答 +20%/层，「弱点再打一拳」

## 七、明确不做的事（防止再次「不酷」）

1. **不堆新页面** — 提高主链路密度  
2. **不做充值 / 掠夺榜** — 家长信任底线  
3. **不做长剧情 / 开放世界** — 一个 Boss、一张海报、一个班级目标够验证  
4. **不靠加题量变酷** — 3 题强反馈 > 30 题平淡  
5. **不把酷建立在纯文案** — 要有立绘、血条、海报、动画  
6. **不把外号绑具体数字** — 绑母题类型  
7. **不在未验证前加第五个系统** — 模块「Boss 化」优于新开系统

## 九、研发协作要点（PM 必知摘要）

| 问题 | 处理 |
|------|------|
| 微信不支持 SVG 立绘 | `npm run gen:boss-png` → PNG |
| 图片路径 | 用 `../../assets/...`，勿用 `/assets/...` |
| Canvas 海报 | `type="2d"`，结算 delay 再绘制 |
| 音效 | 默认静音，课堂场景 |

## 十、会议话术（对齐用）

| 对象 | 话术 |
|------|------|
| **老板** | 不是加功能，是把每日 Boss 做成 5 分钟仪式；外号绑题型，海报做同学接力。 |
| **教研** | 产出「母题类型清单」+ 外号抽检；例题数字可换，类型与外号要稳定。 |
| **设计** | 一张 APNG + 弱点条 + 海报模板，比十张通用插画更值。 |
| **研发** | 优先主链路；模块只接 Boss，不新开系统。 |
| **运营** | 传播抓手是「今天我们班打分糖大盗」，不是活动长文。 |

| 阶段 | 决策 |
|------|------|
| 1 | 诊断：酷 = 3 秒懂 + 30 秒爽 + 可晒物；功能多 ≠ 酷 |
